# One Piece Color Manga — reader

Next.js (App Router) reader for the colorized One Piece manga. Dark theme, mobile +
desktop, SEO-optimized, per-page zoom/pan.

**Live:** https://oneonepiece-color.vercel.app
**Target domain:** https://oneonepiece.colorizedmanga.com (added + verified in Vercel;
needs one DNS record — see below).

## How it works

- Page images live in a separate GitHub repo (`anon6968/op-color-pages`) and are
  served through the **jsDelivr** global CDN. The app never bundles images.
- `data/index.json` + `data/chapters/<n>.json` are the baked catalog (generated by
  `tools/syncdata.py` from the PDF conversion). Every chapter is a static SSG page.
- Image base URL is an env var so it can be repointed to Cloudflare R2 later:

```
NEXT_PUBLIC_SITE_URL=https://oneonepiece.colorizedmanga.com
NEXT_PUBLIC_IMAGE_BASE=https://cdn.jsdelivr.net/gh/anon6968/op-color-pages@main/pages
```

## SEO

- Per-chapter `<title>`/description targeting “colorized one piece manga”, “one piece
  color chapter N”, arc/saga names.
- JSON-LD: WebSite + SearchAction, ComicSeries (home), ComicIssue + BreadcrumbList
  (chapters). Dynamic `sitemap.xml` (every chapter) and `robots.txt`. Canonical URLs,
  OpenGraph/Twitter cards using the chapter cover.

## Finish the custom domain

`oneonepiece.colorizedmanga.com` is already added + verified in the Vercel project.
It only needs DNS (couldn't be done automatically: the provided Cloudflare token is
IP-restricted to another location, and the zone isn't on Vercel):

1. Make sure `colorizedmanga.com`'s nameservers are live on Cloudflare (the apex was
   not resolving at build time).
2. In Cloudflare DNS add: `CNAME  oneonepiece  ->  cname.vercel-dns.com` (DNS-only /
   grey cloud). Or an A record to `216.198.79.1` / `64.29.17.1`.

## Move images to Cloudflare R2 (optional)

Images are on jsDelivr because the R2 S3 endpoint was SNI-blocked from the build
network. To move to R2: run `tools/upload_r2.sh` from any normal network, enable the
bucket's public access / custom domain in Cloudflare, then set
`NEXT_PUBLIC_IMAGE_BASE` to that public base + `/pages` and redeploy.
